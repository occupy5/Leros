# 技术设计 - Leros

> **最后更新：2026-05-21**


## 1. 技能 Schema 定义

技能的技术规格定义：

### 1.1 技能组成

| 字段 | 类型 | 说明 |
|------|------|------|
| 元信息 | Metadata | 名称、描述、版本、分类、标签 |
| 输入Schema | JSON Schema | 技能调用的输入参数定义 |
| 输出Schema | JSON Schema | 技能输出的结果结构定义 |
| 执行流程 | 流程定义 | 技能的内部执行逻辑 |
| 所需工具 | 工具列表 | 技能依赖的工具列表 |
| 所需模型 | 模型规格 | 技能推荐或要求的 AI 模型 |
| 权限声明 | 权限列表 | 技能执行所需的权限（文件读写、网络访问、外部API调用等） |

### 1.2 技能类型

- **工具技能：** 直接调用外部工具或 API 的技能
- **推理技能：** 基于 LLM 的推理能力
- **MCP服务：** 通过标准协议接入的外部服务能力


## 2. 渲染引擎

### 2.1 定位

渲染引擎负责将任务/项目的结构化产出物转换为目标格式文件。

### 2.2 支持的输出格式

- Word（.docx）
- PDF
- Markdown
- HTML

### 2.3 核心能力

- **模板匹配与样式渲染：** 从任务产出设置中选择模板，按模板定义的样式进行渲染
- **预览与调整：** 用户导出前确认内容与样式
- **批量导出：** 项目级交付物打包导出


## 3. 技能模板（技术规格）

### 3.1 模板结构

模板是任务产出的格式化配置，包含：

| 组成部分 | 说明 |
|---------|------|
| 结构模板 | 章节组织、段落占位、目录 |
| 格式模板 | 字体、字号、页眉页脚、样式表 |
| 变量定义 | 可填充变量（如 `{{客户名称}}`、`{{项目编号}}`） |
| 变量映射 | 从技能输入 Schema 自动匹配变量来源 |

### 3.2 模板存储

- 模板以结构化格式（JSON/YAML）存储
- 关联渲染引擎的样式定义
- 支持版本管理
