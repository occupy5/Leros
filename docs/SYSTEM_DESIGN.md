# 系统架构设计 - Leros

> **最后更新：2026-05-21**


## 1. 核心实体关系

```
项目（长期上下文）
 ├── 会话（scope=project）
 ├── 任务
 │     └── 会话（scope=task）
 ├── AI队友
 ├── 知识库
 └── 项目资产

独立任务（临时，无项目）
 └── 会话（scope=standalone）

平台全局
 ├── 全局技能库
 ├── 全局知识库
 └── AI队友运行时
```


## 2. 知识检索架构

### 2.1 检索方式

**全文检索：** 基于关键词的全文搜索。

**向量检索：** 基于语义的向量相似度搜索。

**混合检索：** 全文检索与向量检索结合。

**图谱检索：** 基于知识图谱的关系检索。

**RAG 集成：** 与 AI队友执行环境集成的检索增强生成。


## 3. 平台引擎

### 3.1 定位

平台引擎为 AI队友提供真实的工作环境，包括 AI 模型、执行节点、外部连接和基础设施。

### 3.2 AI 模型管理

**模型供应商管理：** 支持多家模型供应商和本地部署模型。

**模型目录：** 维护模型名称、能力标签、上下文长度、成本、延迟、适用场景等信息。

**凭证管理：** 管理 API 密钥、组织 ID 等凭证信息，支持密钥轮换。

**路由策略：** 按 AI队友、任务类型、成本预算选择合适的模型。

**降级策略：** 主模型失败后切换备用模型。

**调用审计：** 记录模型调用、token 用量、耗时、错误和成本。

### 3.3 工作节点

工作节点是 AI队友执行真实任务的基础设施。

**支持的节点类型：**
- 容器沙箱节点：基于容器技术的隔离执行环境
- 远程执行节点：连接到远程服务器执行任务
- Browser 节点：浏览器自动化执行环境（规划中）
- Code Runtime：代码执行运行时（规划中）

**节点能力：**
- 节点健康检查
- 资源容量管理（CPU、内存、磁盘）
- 工作目录管理
- 资源限制
- 执行审计（命令日志、文件变更、执行产物）
- 沙箱生命周期管理

### 3.4 外部连接

外部连接解决"外部系统如何触发任务"的问题。按连接器类型进行差异化配置与管理。

#### 3.4.1 连接器类型

**代码平台连接器：**
- 平台：GitHub、GitLab、Gitee 等
- 事件：PR/MR 创建、更新、评论，Issue 创建、评论
- 能力：代码读取、评论回写、状态设置

**CI/CD 连接器：**
- 平台：Jenkins、GitHub Actions、GitLab CI 等
- 事件：构建完成、部署完成、流水线失败
- 能力：触发流水线、查询构建状态、获取构建产物

**数据源连接器：**
- 监控：Prometheus、Zabbix 等
- 日志：ELK、Loki 等
- APM：SkyWalking、Datadog 等
- 能力：数据查询、指标订阅、告警拉取

**业务系统连接器：**
- 系统：工单系统、审批系统、告警系统等
- 事件：工单创建、状态变更、审批完成、告警触发/恢复
- 能力：内容映射、状态回写

#### 3.4.2 通用连接能力

所有连接器类型共享以下通用能力：
- 事件类型配置：选择订阅哪些类型的事件
- 请求签名和密钥管理：验证请求来源的合法性
- 内容映射：将外部事件内容映射为平台内部任务
- 幂等处理：避免重复处理同一事件
- 失败重试：事件处理失败时自动重试

### 3.5 IM 身份

IM 身份用于将 AI队友绑定到企业即时通讯工具中的机器人。

**支持的 IM 渠道：** 企业微信、飞书、钉钉、Slack 等。

**典型能力：**
- 私聊消息
- 群聊消息
- @ 消息触发
- 卡片消息
- 文件消息
- 身份映射
- 消息审计


## 4. 上下文引擎（规划）

长会话（特别是 scope=project 的长期会话）随项目推进内容会持续膨胀。上下文引擎负责会话上下文的管理与优化。

**规划能力：**
- 上下文裁剪：自动移除冗余和低价值内容
- 记忆摘要：将长对话压缩为结构化摘要
- 历史检索：基于向量检索回溯历史会话内容
- Checkpoint：关键节点保存会话快照

> 当前阶段作为规划方向记录，不在 MVP 范围内。
