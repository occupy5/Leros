# 产品需求文档 (PRD) - Leros

> **最后更新：2026-05-21**
> **版本：3.0**


## 1. 产品定位

Leros 是一个面向企业与团队的 AI 工作协作系统（AI Workspace）。

系统通过以下核心要素构建可持续协作、可执行、可审计的 AI 工作环境：

- **项目（Project）** — 长期工作的核心容器
- **会话（Session）** — 实时协作线程
- **任务（Task）** — 最小执行单元
- **AI队友（Agent）** — 人格化执行者
- **技能（Skill）** — 可复用的能力模块
- **知识库（Knowledge）** — 长期记忆与上下文
- **项目资产（Artifacts）** — 交付物集合

Leros 不只是聊天工具。它更像：

- AI Native 的 Jira — 以任务驱动协作
- AI Native 的 Notion — 以知识组织工作
- AI Native 的 Cursor Team — 人机协同开发
- AI Native 的自动化工作系统 — 可持续执行的 AI 流水线

用户可以像组织真实团队一样：创建 AI 队友、分配任务、绑定技能、共享知识、跟踪项目、审计执行过程 — 让 AI 真正参与长期工作流，而不是一次性对话。


## 2. 产品核心概念

### 2.1 核心业务模型

```
项目（Project）→ 长期上下文
 ├── 会话（Session）→ 协作线程（可多个）
 ├── 任务（Task）→ 最小执行单元
 │     └── 会话（Session）→ 执行上下文
 ├── AI队友（Agent）→ 人格化执行者
 ├── 知识库（Knowledge）→ 长期记忆
 └── 项目资产（Artifacts）→ 交付物集合

执行流：
会话（协作讨论）
  ↓  讨论成熟，创建任务
任务
  ↓
会话（执行上下文）
  ↓
项目资产（产出物）
  ↓  沉淀
知识库 / 项目记忆
```

### 2.2 核心实体

| 实体 | 定义 | 职责 |
|------|------|------|
| 项目（Project） | 长期工作容器 | 组织会话、任务、AI队友、知识、资产 |
| 会话（Session） | 协作线程 | 承载人类与 AI 的实时协作与上下文交流。scope=project 为项目协作，scope=task 为任务执行，scope=standalone 为独立对话 |
| 任务（Task） | 最小执行单元 | 定义要做什么，谁来做，输出什么。project_id 可空 |
| 项目资产（Artifacts） | 交付物集合 | 聚合管理项目所有任务产出与交付件 |
| AI队友（Agent） | 人格化执行者 | 理解任务、调用技能、产出结果 |
| 技能（Skill） | 可复用能力模块 | 定义输入、输出、执行流程 |
| 知识库（Knowledge） | 持久化记忆 | 提供领域知识、项目上下文 |

### 2.3 协作关系

**执行流程：**

```
会话（协作讨论，scope=project）
  ↓  讨论成熟，创建任务
创建任务
  ↓
会话（执行上下文，scope=task）
AI队友调用技能、工具
  ↓
项目资产（产出物：文档/代码/报告）
  ↓  沉淀
知识库 / 项目记忆
```

### 2.4 与传统工具的对应

| Leros | 类比 |
|-------|------|
| 项目 | Jira Project + Notion Page |
| 任务 | Jira Issue + GitHub Issue |
| AI队友 | 团队成员（PM/Dev/QA） |
| 技能 | 工具 + API |
| 知识库 | Confluence + 代码文档 |


## 3. 产品信息架构（IA）

### 3.1 一级导航结构

```
工作台
任务
技能
知识库
项目
AI队友
个人中心
```

### 3.2 导航定义

#### 工作台

用户的统一工作入口。

展示：
- 最近任务
- 待处理事项
- AI 协作动态
- 最近项目
- AI队友状态
- 快捷创建入口

#### 任务

任务是系统中的最小执行单元。

用户可以：
- 创建任务
- 指派 AI队友
- 跟踪执行状态
- 查看执行过程
- 接管任务
- 审核结果

#### 技能

技能是 AI 的可复用能力模块。

例如：PR审查、文档撰写、SQL分析、数据总结、Web搜索、CAD分析。

技能可以被：
- AI队友调用
- 任务直接调用

#### 知识库

知识库用于提供长期上下文与企业知识。

支持：文档知识、代码知识、项目知识、FAQ、向量检索、图谱检索。

#### 项目

项目用于组织长期任务、上下文和协作关系。

项目包含：项目目标、长期上下文、任务列表、AI队友、项目知识、项目记忆。

项目是长期工作的核心容器。

#### 个人中心

个人中心是用户管理个人配置和偏好的入口。

包含：
- **个人信息：** 头像、昵称、联系方式
- **账号安全：** 密码修改、登录记录、多因素认证
- **API 密钥：** 管理个人 API Token，用于外部系统集成
- **通知偏好：** 配置消息通知的渠道和频率
- **界面偏好：** 主题、语言、时区等个性化设置
- **操作日志：** 个人操作历史记录
- **使用统计：** 个人任务量、资源使用情况概览


## 4. 工作台

### 4.1 定位

工作台是用户进入系统后的统一工作入口，聚合展示用户关注的核心信息，提供快捷操作入口。

### 4.2 核心模块

**最近任务：** 展示当前用户最近创建或指派的待处理任务。

**待处理事项：** 需要用户审批、验收或接管的任务列表。

**AI协作动态：** 系统内 AI队友的执行动态，包括任务完成、审批请求、异常事件等。

**最近项目：** 用户最近参与的项目列表，支持快速跳转。

**AI队友状态：** 用户常用的 AI队友当前在线状态和忙碌情况。

**快捷创建入口：** 快速创建任务、项目或对话。

### 4.3 个性化

- 支持用户自定义工作台布局
- 支持 Pin 住常用项目和 AI队友
- 支持设置通知偏好


## 5. 任务系统

### 5.1 定位

任务是系统中的最小执行单元。任务驱动 AI队友工作，是人和 AI 协作的基本单位。

### 5.2 任务生命周期

```
创建 → 待分配 → 执行中 → 待审批 → 完成
                    → 失败
                    → 已取消
                    → 已暂停
```

**创建：** 用户创建任务，填写任务描述、目标、预期输出。

**待分配：** 任务等待分配 AI队友或人工执行。

**执行中：** AI队友正在处理任务，或人工正在执行。

**待审批：** 任务执行到需要人工审批的节点，暂停等待审批。

**完成：** 任务成功完成，结果已输出。

**失败：** 任务执行失败，可能由于错误、超时等原因。

**已取消：** 任务被用户取消。

**已暂停：** 任务被用户临时暂停，可恢复。

### 5.3 会话（Session）

任务执行时自动创建 `scope=task` 的会话，作为任务的执行上下文，记录从开始到完成的完整过程。

**任务属性中的关联会话（参考 5.5）：**
- `session_id`：关联的执行会话

**会话内容：**
- **执行对话：** AI队友与用户的交互记录
- **工具调用：** 调用的技能、工具及参数
- **中间结果：** 执行过程中的阶段性产出
- **决策记录：** 关键决策点和选择依据

**会话特性：**
- 自动创建：任务开始执行时自动创建
- 可回溯：支持回放执行过程和审计
- 可继续：暂停/恢复任务时延续会话

### 5.4 任务类型

**普通任务：** 一次性执行的任务。用户创建任务并指派 AI队友，AI队友调用相应技能完成。任务的具体工作内容（文档撰写、代码编写、数据调研等）由绑定的技能定义，而非任务类型。

**周期任务：** 按计划周期性执行的任务。详见 5.7 周期任务。

### 5.5 任务属性

**优先级：** 任务的优先级，影响执行顺序（紧急/高/中/低）。

**截止时间：** 任务需要完成的时间要求。

**标签：** 用于分类和筛选任务的标签。

**关联会话：** 任务关联的执行会话（scope=task）。

**关联项目：** 任务所属的项目。独立任务（临时场景）可无关联项目，project_id 为空。

**关联 AI队友：** 执行任务的 AI队友。

**子任务：** 一个任务可以拆分为多个子任务。

**依赖关系：** 任务之间的依赖关系。

**任务上下文：** 任务相关的背景信息、参考资料。

**预期输出：** 任务完成后的产出物描述。

### 5.6 人机协作

**AI执行：** 默认由 AI队友自动执行任务。

**用户接管：** 用户可以随时接管任务，转为人工执行。

**用户审批：** 关键节点需要用户审批后才能继续。

**用户补充上下文：** 用户可以在任务执行过程中补充信息。

**用户中断：** 用户可以中断正在执行的任务，修改指令后继续。

**用户恢复：** 中断或暂停的任务可以恢复执行。

### 5.7 周期任务

支持周期性任务配置（Cron 表达式），用于：
- 每日项目进展摘要
- 定期代码仓库巡检
- 定时知识库同步
- 定时数据分析
- 定时运营报告生成

### 5.8 任务资产（Task Artifacts）

任务执行完成后产生可交付的产出物。任务资产是项目资产的组成部分。

**产出类型：**
- **文本内容：** 文档正文、代码、报告等文本形式的输出
- **结构化数据：** JSON、表格、CSV 等结构化输出
- **文件附件：** 图片、压缩包、二进制文件等
- **复合产物：** 包含多种类型的综合交付物

**导出能力：**
- 支持导出 Word、PDF、Markdown、HTML 格式
- 预览与调整（用户导出前确认内容与样式）

**模板（可选）：**
- 模板是任务产出的格式化配置，定义文档结构和样式
- 创建任务时在产出设置中选择模板，也可留空使用默认格式
- 技能中心可推荐模板并说明适用场景
- 模板独立于技能，同一技能可配合不同模板使用

**产出管理：**
- 产出溯源（关联到执行过程中的技能调用和知识引用）
- 版本保留（每次迭代保留历史版本，可回溯）
- 纳入项目知识库（任务产出可沉淀为项目知识）


## 6. 项目系统

### 6.1 定位

项目是系统中组织长期工作的一级核心实体。

项目承载：长期目标、上下文、协作关系、任务、AI队友、知识。

项目是 AI 长期记忆和持续协作的核心容器。

### 6.2 项目组成

```
项目（长期上下文）
 ├── 项目信息（名称、描述、目标、状态）
 ├── 会话（scope=project）
 ├── 任务列表
 │     └── 每个任务 → 会话（scope=task）
 ├── AI队友（绑定多个 AI队友，不同角色）
 ├── 知识库（项目专属知识）
 ├── 项目资产（所有交付物的聚合）
 ├── 项目记忆（系统自动沉淀）
 └── 项目成员（参与的人类用户）
```

### 6.3 项目能力

**项目上下文：** 维护长期背景信息，包括商业目标、技术架构、项目规范、业务资料。

**项目任务：** 项目下可创建多个任务，任务共享项目上下文。

**项目AI队友：** 项目可以绑定多个 AI队友，例如 PM、架构师、开发、测试、安全审计。

**项目知识库：** 项目可以拥有独立的知识库，存储项目相关的文档、代码、规范。

**项目记忆：** 系统自动沉淀历史决策、任务结果、讨论过程、执行经验。

**项目模板：** 支持将项目保存为模板，快速创建同类项目。

### 6.4 会话（Session）

项目级会话是 `scope=project` 的会话，是项目内的长期协作线程，提供项目成员（含 AI队友）之间的持续讨论空间。

每个项目有一个会话，项目创建时自动生成。

**与任务会话的区别：**
- 项目会话不绑定具体输出目标，是自由讨论和规划空间
- 任务会话绑定到具体任务，记录执行过程
- 项目会话上下文持久化，延续整个项目生命周期

**典型用途：**
- 在项目成员间讨论项目方向、方案设计
- 向 AI队友咨询项目相关问题
- 回顾项目历史决策和讨论过程

**会话管理：**
- 从项目会话中可以创建任务：协作讨论成熟 → 创建任务 → 自动关联任务会话
- 会话内容自动归入项目记忆
- 支持搜索历史会话内容

### 6.5 项目生命周期

```
创建 → 进行中 → 暂停 → 已完成 → 已归档
```

- **创建：** 填写项目信息，设定目标和范围
- **进行中：** 项目活跃，任务正在执行
- **暂停：** 项目暂时中止，可恢复
- **已完成：** 项目目标已达成
- **已归档：** 项目关闭，数据保留只读

### 6.6 项目资产（Project Artifacts）

项目资产是项目下所有任务资产和交付物的聚合视图。

**聚合管理：**
- 按任务维度查看项目所有产出物
- 按类型筛选（文档、代码、报告等）
- 产出物检索与全文搜索

**交付管理：**
- 项目级交付物打包导出（全部或选定的产出物）
- 交付物清单与版本标记
- 外部交付物登记（上传外部交付物关联项目）

**知识沉淀：**
- 项目产出自动纳入项目知识库
- 产出物可作为后续任务的上文参考
- 优秀产出可提升为团队模板


## 7. AI队友系统

### 7.1 定位

AI队友是可长期协作的 AI 执行角色。AI队友不再是平台中心，而是任务执行角色。

### 7.2 AI队友核心组成

```
AI队友 = 角色 + 技能 + 记忆 + 执行环境
```

### 7.3 角色（Role）

**基础信息：** 名称、头像、描述。

**职责定义：** 负责的工作领域，例如"代码审查"、"问题解答"、"数据分析"。

**工作边界：** 不能做的事情，例如"不允许修改生产环境配置"、"不允许访问敏感数据"。

**系统提示词：** 行为准则和回复风格。

**输出格式：** 输出结果的格式要求。

**拒绝策略：** 超出职责范围或违反边界时的回应方式。

**审批场景：** 哪些操作需要人工审批后才能执行。

### 7.4 技能（Skill）

AI队友绑定的可执行能力模块。

**绑定方式：**
- 从全局技能库选取
- 从项目技能库选取
- 创建自定义技能

**技能权限：** 控制 AI队友对每个技能的使用权限（允许/禁止/需审批）。

### 7.5 记忆（Memory）

AI队友可访问的上下文。

**记忆来源：**
- 项目记忆（当前参与的项目）
- 知识库（已授权的知识库）
- 历史任务（之前执行过的任务）
- 长期偏好（用户设定的偏好）

### 7.6 执行环境（Runtime）

AI队友实际工作的运行环境。

**执行环境配置：**
- 默认模型
- 可用工具
- 工作节点
- 网络策略
- 资源限制

## 8. 技能系统

### 8.1 定位

技能是平台中的标准能力单元，是平台的核心资产。

技能定义：输入结构、输出结构、所需工具、所需模型、执行流程、权限需求。

### 8.2 技能分类

| 分类 | 说明 | 来源 |
|------|------|------|
| 系统技能 | 平台内置的标准能力 | 平台提供 |
| 项目技能 | 项目私有的能力 | 项目创建 |
| 团队技能 | 组织共享的能力 | 团队创建 |
| 市场技能 | 来自第三方的能力 | 技能市场 |

### 8.3 技能组成

**元信息：** 名称、描述、版本、分类、标签。

**输入输出定义：** 技能调用的输入参数和输出结果的结构定义。

**执行流程：** 技能的内部执行逻辑。

**所需工具：** 技能依赖的工具列表。

**所需模型：** 技能推荐或要求的 AI 模型。

**权限声明：** 技能执行所需的权限（文件读写、网络访问、外部API调用等）。

### 8.4 技能类型

**工具技能：** 直接调用外部工具或 API 的技能，如 Web搜索、文件操作。

**推理技能：** 基于 LLM 的推理能力，如代码审查、文本摘要、分类判断。

**MCP服务：** 通过标准协议接入的外部服务能力。

### 8.5 技能管理

- 技能注册：注册新的技能
- 技能分类：按功能领域分类
- 版本管理：管理技能的版本和变更
- 依赖声明：声明技能的依赖关系
- 安全审核：对技能进行安全审核
- 启用和停用：控制技能的可用性
- 使用统计：统计技能的使用频率和效果


## 9. 知识库系统

### 9.1 定位

知识库为 AI队友提供企业内部上下文信息，是 AI 长期记忆的持久化存储。

### 9.2 知识类型

**文档知识：** 技术文档、产品文档、规范文档等。

**代码知识：** 代码仓库结构、API 定义、代码规范等。

**项目知识：** 项目目标、架构决策、业务资料等。

**FAQ：** 常见问题与答案。

**外部知识：** 通过连接器获取的外部知识源。

### 9.3 知识管理

**知识导入：** 从文档、代码仓库、数据库等来源导入知识。

**知识分类：** 按领域、项目、部门等维度分类。

**知识更新：** 支持增量更新和自动同步。

**权限控制：** 控制不同 AI队友和用户可访问的知识范围。

**使用统计：** 统计知识的使用情况。

### 9.4 知识检索
